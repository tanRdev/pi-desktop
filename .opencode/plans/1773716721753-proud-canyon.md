# PiDesk Desktop Production Hardening Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Isolation rule:** REQUIRED before any code changes: use superpowers:using-git-worktrees and execute in a global worktree at `~/.config/superpowers/worktrees/PiDesk/desktop-production-hardening` (explicit assumption chosen because the current repo is dirty, no project-local worktree directory exists, and `.gitignore` does not currently protect one).
>
> **Artifacts rule:** Store baseline logs, Playwright traces, and temporary test user data outside the repo at `~/.config/superpowers/worktrees/PiDesk/_artifacts/desktop-production-hardening/` and retain them until the user accepts the work.

**Goal:** Make the desktop app production-ready by eliminating high-risk state, IPC, terminal, and UI defects; splitting megafiles into smaller test-backed modules; and polishing the existing interface without changing the product direction.

**Architecture:** Keep the current Electron + React canvas/workspace model, but harden it in thin TDD slices. Extract deterministic logic into small modules first, turn `app.tsx`, `ipc-router.ts`, `terminal-manager.ts`, and selected `index.ts` helpers into orchestration shells, and use the existing Vitest integration + Playwright E2E setup as the primary safety net.

**Tech Stack:** Electron 41, React 19, TypeScript, Tailwind CSS 4, pnpm workspace, Vitest workspace integration tests, Playwright E2E.

---

## Assumptions

- The current dark desktop UI direction is correct; this effort refines hierarchy, accessibility, responsiveness, and polish rather than redesigning the product.
- The existing dirty working tree is out of scope and must remain untouched.
- No framework rewrite, Electron replacement, or large dependency-refresh campaign is allowed inside this effort.
- Existing integration/E2E suites are the starting conventions; prefer expanding them over introducing a second testing paradigm unless current coverage proves insufficient.
- If in-repo CI is required, GitHub Actions is the default assumption because no existing `.github/workflows/` directory was found.
- Node `>=24.13.1`, pnpm `10.32.1`, and native build prerequisites for `electron-rebuild`/`node-pty` must be available in the isolated workspace before any implementation starts.
- Playwright E2E launches the built Electron artifact at `apps/desktop/out/main/index.js`, so every E2E run must happen after a fresh desktop build.

## Scope

- TDD-first hardening of renderer state management, main-process IPC, terminal lifecycle, and multi-window/thread flows.
- Megafile decomposition for:
  - `apps/desktop/src/renderer/src/app.tsx`
  - `apps/desktop/src/main/ipc-router.ts`
  - `apps/desktop/src/main/terminal-manager.ts`
  - selected helper extraction from `apps/desktop/src/main/index.ts`
- DRY cleanup for repeated window math, prompt-routing/window-opening logic, IPC payload parsing, and terminal event wiring.
- Accessibility, interaction, and visual polish for the desktop shell, sidebar, title bar, prompt area, and canvas.
- Release-readiness gates for lint, typecheck, integration tests, E2E, build, perf scripts, and React/UI audits.

## Non-goals

- No product-scope expansion unrelated to hardening/polish.
- No rewrite of the canvas model, shell model, or Electron app lifecycle from scratch.
- No destructive migration of persisted repository/thread/window state.
- No automatic commits, pushes, or PR creation unless the user explicitly asks during build mode.

## Structural targets

- `apps/desktop/src/renderer/src/app.tsx`: reduce from ~1600 lines to an orchestration shell plus extracted modules/components; target final file size ≤ 700 lines.
- `apps/desktop/src/main/ipc-router.ts`: reduce from ~580 lines to a thin registrar/composition layer; target final file size ≤ 220 lines.
- `apps/desktop/src/main/terminal-manager.ts`: reduce from ~400 lines to an orchestration shell with extracted backend/session helpers; target final file size ≤ 250 lines.
- `apps/desktop/src/main/index.ts`: keep `bootstrapDesktop()` as the composition root, but extract transport/model-switch/thread-context helpers until it is primarily startup orchestration.

## Risks and mitigations

- **Persisted-state regressions:** add/expand tests around selection/worktree/thread restoration before changing persistence-related code; do not rename keys without migration handling.
- **Terminal orphan processes:** lock behavior with terminal lifecycle tests before extraction; verify `destroyAll()` and exit cleanup explicitly.
- **UI regressions during polish:** use focused Playwright smoke + accessibility tests after each UI chunk rather than one large visual rewrite.
- **E2E flakiness:** fix readiness signaling/root cause first; do not paper over timing issues with arbitrary waits.
- **Dirty-tree contamination:** perform all execution in the isolated global worktree; never stage or edit the original working tree.
